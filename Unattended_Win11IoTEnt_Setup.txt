INTRODUCTION	 

Windows Setup can be automated using XML files. Creating, editing, and managing these files is best done using the Windows System Image Manager (WSIM). This document details the settings used for our customized ECHO and Wafer Windows 10 IoT Enterprise images. 

NOTE: Because these are XML files it is very tempting to simply modify them manually, but this can be very risky! The Answer File format has very strict rules about the format of data for each element AND whether a given element is allowed to be empty. Certain Components / Settings can only be applied to certain Passes, also. It is highly recommended to use WSIM to ensure compliance and keep the Answer File valid. 



Understanding windows 11 Setup 

Windows 11 Setup is designed to be executed across several different “Passes”. There are seven different Passes of the installation where settings from an unattended file can be applied. These stages are: 

windowsPE 

These settings are used during the pre-installation phase (windowsPE is the Windows Pre-Installation Environment). 

offlineServicing 

These settings are applied when doing offlineServicing to an image. 

Generalize 

These settings are used when the sysprep tool generalizes an image prior to capture. 

Specialize 

These settings are used when an image is being deployed. 

auditSystem 

These settings are used for System specific options applied when the system is put into Audit mode. 

audit User 

These settings are used for User specific options applied when the system is put into Audit mode. 

oobeSystem 

These settings are used during the Out-Of-Box-Experience (OOBE) which would run when a customer first takes the PC out of the box. Since our PCs are already embedded in tools, we have decided to automate through OOBE to simplify PC setup for the Production Team. 

 

Getting Started with WSIM 

WSIM is a powerful tool with many different options. This document will only detail specifically what we’re using to automate our customized Windows 10 IoT Enterprise image installations. For anything outside of the scope of this document you’ll need to research and try it yourself. 

Before you can configure an Answer File you must first select an existing Windows image (WIM file).  

Go to File…Select Windows Image… 

Choose the WIM file for which you’d like to configure an Answer File. 

If this is the first time you’ve chosen, this WIM file WSIM will notify you that you must have a valid catalog file to continue and ask if you’d like to create one. Choose “Yes”. 

If this is NOT the first time you’ve opened this WIM file or you’re opening an existing Answer File (which will load it’s associated WIM) WSIM will check that the existing catalog is still valid. If WIM has changed since the last time the catalog was generated, you’ll be notified that the catalog is out of date. Choose “Yes” to re-create the catalog. 

Creating the catalog can take some time depending on the size of your WIM. 

Once the catalog is created you can now continue and create / edit your Answer File.

Adding Components / Settings to the Answer File 

The lower left of the WSIM window is a pane for the currently loaded Windows Image (WIM):

Expand the Components Folder to explore all the different Components and their accompanying settings. 

There are a LOT of different components, consult the Help menu of WSIM (redirects to Microsoft website for Unattended Windows Setup Reference) for more information. 

One thing to note: 

The Components prefixed with “amd64” are used with 64-bit Windows. You should always be using the “amd64” prefixed Components. 

The Components prefixed with “wow64” are used with WoW64 (commonly referred to as Windows-32-On-Windows-64) which is the 32-bit Windows compatibility layer within 64-bit Windows. There should be no reason to use “wow64” prefixed Components. 

To add a new Component / Setting: 

Explore the Components Folder to find the desired Component / Setting. 

Right-click on the Component / Setting. A context menu will appear where you will choose which Pass to add this Setting to. This menu will only allow you to choose a Pass where the setting will be valid.
